# Практическое задание 35.1

## Задания

### Описание задачи
На предыдущих уроках ты изучил коллекции (списки, словари, множества) и базовые алгоритмы для работы с ними. Теперь мы начинаем изучать функции. Этот блок посвящен основам: объявлению функций, передаче аргументов и возврату результата. Твоя задача — научиться создавать простые функции, которые принимают данные, выполняют вычисления и возвращают результат, а не просто печатают его. Это фундаментальный навык, который позволит в будущем структурировать код и избегать дублирования.

Контекст: ты пишешь модуль для анализа данных о продажах в небольшом магазине. Данные представлены в виде списков. Тебе нужно реализовать несколько функций для базовых расчетов.

### Пример входных и выходных данных
**Пример 1: Расчет общей суммы продаж**
*   **Входные данные:**
    *   Список продаж за день: `sales = [150, 200, 75, 300, 100]`
*   **Что нужно сделать:** Написать функцию `calculate_total`, которая принимает список чисел (продаж) и возвращает их сумму.
*   **Ожидаемый результат:** `calculate_total(sales)` возвращает `825`.
*   **Пояснение:** Функция должна пройти по всем элементам списка, сложить их и вернуть итог. Не используй встроенную `sum()` для реализации логики сложения (используй цикл), но можешь использовать ее для проверки.

**Пример 2: Поиск максимальной продажи**
*   **Входные данные:**
    *   Список продаж: `sales = [150, 200, 75, 300, 100]`
*   **Что нужно сделать:** Написать функцию `find_max_sale`, которая принимает список чисел и возвращает наибольшее значение.
*   **Ожидаемый результат:** `find_max_sale(sales)` возвращает `300`.
*   **Пояснение:** Функция должна сравнить все элементы списка и найти максимальный. Не используй встроенные `max()` или сортировку для реализации логики поиска.

**Пример 3: Подсчет продаж выше порога**
*   **Входные данные:**
    *   Список продаж: `sales = [150, 200, 75, 300, 100]`
    *   Пороговое значение: `threshold = 120`
*   **Что нужно сделать:** Написать функцию `count_above_threshold`, которая принимает список чисел и пороговое значение. Функция должна вернуть количество продаж, сумма которых превышает порог.
*   **Ожидаемый результат:** `count_above_threshold(sales, threshold)` возвращает `3` (продажи 150, 200 и 300 превышают 120).
*   **Пояснение:** Функция должна проверить каждый элемент списка и увеличивать счетчик, если значение больше порога.

### Критерии проверки и ограничения
1.  **Обязательно:** Каждая задача должна быть решена в виде отдельной функции, объявленной с помощью `def`.
2.  **Обязательно:** Функции должны принимать аргументы (параметры) и возвращать результат с помощью оператора `return`. Запрещено использовать `print` внутри функции для вывода результата расчета.
3.  **Обязательно:** Используй только позиционные аргументы. Темы "именованные аргументы" и "значения по умолчанию" еще не изучены.
4.  **Запрещено:** Использовать встроенные функции `sum()`, `max()`, `sorted()` для реализации основной логики в функциях `calculate_total` и `find_max_sale`. Их можно использовать только для проверки корректности твоего кода вне этих функций.
5.  **Запрещено:** Использовать темы из будущих блоков: `*args`, `**kwargs`, `global`, изменяемые значения по умолчанию.
6.  **Проверка:** Твой код должен корректно работать с любыми списками чисел (включая пустые списки для функций, где это логично). Подумай, что должна вернуть функция `find_max_sale` для пустого списка?

### Решение задачи (псевдокод)
Для каждой из трех функций опишем шаги решения.

**Функция calculate_total(sales_list):**
1.  Объявить переменную-накопитель `total_sum` и установить ее в 0.
2.  Для каждого элемента `sale` в `sales_list`:
    *   Прибавить значение `sale` к `total_sum`.
3.  Вернуть значение `total_sum`.

**Функция find_max_sale(sales_list):**
1.  Если `sales_list` пуст:
    *   Вернуть `None` (так как максимального значения не существует).
2.  Объявить переменную `current_max` и установить ее равной первому элементу `sales_list`.
3.  Для каждого оставшегося элемента `sale` в `sales_list`:
    *   Если `sale` больше `current_max`:
        *   Обновить `current_max`, присвоив ему значение `sale`.
4.  Вернуть значение `current_max`.

**Функция count_above_threshold(sales_list, limit):**
1.  Объявить переменную-счетчик `count` и установить ее в 0.
2.  Для каждого элемента `sale` в `sales_list`:
    *   Если `sale` больше `limit`:
        *   Увеличить `count` на 1.
3.  Вернуть значение `count`.

### Рекомендации
*   **Вынести в отдельную функцию:** Каждая из трех описанных задач уже является отдельной функцией. Это и есть цель задания — практика создания функций. Не пытайся объединить логику всех трех расчетов в одну функцию.
*   **PEP8:**
    *   Используй осмысленные имена для функций и переменных (`calculate_total` вместо `func1`, `sales_list` вместо `lst`).
    *   После объявления функции `def` и перед ее телом оставляй одну пустую строку. Между самими функциями оставляй две пустые строки для лучшей читаемости.
