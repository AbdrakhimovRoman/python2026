# Практическое задание 35.3

## Задания

### Описание задачи
В предыдущих блоках ты научился работать с позиционными и именованными аргументами, а также значениями по умолчанию. Теперь мы переходим к созданию гибких функций, которые могут работать с переменным количеством данных. Это критически важно для написания универсального кода, способного адаптироваться к заранее неизвестному количеству входных данных, что часто встречается в реальных задачах.

Твоя задача — создать функцию-агрегатор `collect_stats`, которая собирает статистику по произвольному набору числовых данных, переданных разными способами. Функция должна уметь:
1.  Принимать любое количество позиционных аргументов (числа) через `*args`.
2.  Принимать любое количество именованных аргументов (пары "метка-число") через `**kwargs`.
3.  Возвращать словарь с рассчитанной статистикой: общее количество переданных чисел, их сумму, максимальное и минимальное значение.
4.  Корректно обрабатывать ситуацию, когда функция вызвана без аргументов.

Для проверки работы функции тебе также нужно будет написать вспомогательную функцию `unpack_and_call`, которая демонстрирует технику распаковки. Эта функция должна принимать список и словарь, распаковывать их и передавать аргументы в `collect_stats`.

### Пример входных и выходных данных

**Пример 1: Прямой вызов с позиционными и именованными аргументами**
```python
result = collect_stats(10, 20, 30, temp=15, pressure=1013)
print(result)
```
**Ожидаемый вывод:**
```python
{
    'count': 5,
    'sum': 1088,
    'max': 1013,
    'min': 10
}
```
*Пояснение:* Функция получила три позиционных аргумента (10, 20, 30) и два именованных (`temp=15`, `pressure=1013`). Всего 5 чисел. Сумма: 10+20+30+15+1013 = 1088. Максимум — 1013, минимум — 10.

**Пример 2: Вызов с распаковкой из `unpack_and_call`**
```python
my_list = [5, 15, 25]
my_dict = {'reading': 100, 'offset': -5}
result = unpack_and_call(my_list, my_dict)
print(result)
```
**Ожидаемый вывод:**
```python
{
    'count': 5,
    'sum': 140,  # 5+15+25+100+(-5)
    'max': 100,
    'min': -5
}
```
*Пояснение:* Функция `unpack_and_call` должна распаковать список `my_list` в позиционные аргументы, а словарь `my_dict` — в именованные, и вызвать `collect_stats(5, 15, 25, reading=100, offset=-5)`.

**Пример 3: Вызов без аргументов**
```python
result = collect_stats()
print(result)
```
**Ожидаемый вывод:**
```python
{
    'count': 0,
    'sum': 0,
    'max': None,
    'min': None
}
```
*Пояснение:* Поскольку чисел нет, сумма и количество равны 0, а максимум и минимум не определены (`None`).

### Критерии проверки и ограничения
1.  Функция `collect_stats` должна быть объявлена с использованием `*args` и `**kwargs`.
2.  Все числовые данные из `args` (кортеж) и `kwargs` (словарь) должны быть учтены в расчетах.
3.  Для вычисления суммы, максимума и минимума используй встроенные функции `sum()`, `max()` и `min()`. Учти, что они не работают с пустыми последовательностями.
4.  Функция `unpack_and_call` должна принимать два параметра (список и словарь) и внутри себя вызывать `collect_stats`, используя операторы распаковки `*` и `**`.
5.  Не используй циклы `for` для обхода `args` или `kwargs` — применяй преобразование в список/кортеж и встроенные функции агрегации.
6.  **Важное ограничение:** В этом задании запрещено использовать темы из будущих блоков, такие как область видимости, глобальные переменные (`global`), замыкания. Работай только с передачей и распаковкой аргументов.

### Решение задачи (псевдокод)

**Для функции collect_stats(*args, **kwargs):**
1.  Объединить все числовые значения в один общий список:
    *   Преобразовать `args` (это кортеж) в список.
    *   Получить все значения из словаря `kwargs`.
    *   Создать новый список, объединив эти два набора значений.
2.  Вычислить количество элементов в объединенном списке.
3.  Если список не пустой:
    *   Вычислить сумму элементов списка.
    *   Найти максимальный элемент в списке.
    *   Найти минимальный элемент в списке.
4.  Если список пустой:
    *   Установить сумму равной 0.
    *   Установить максимум и минимум в `None`.
5.  Вернуть словарь с ключами `'count'`, `'sum'`, `'max'`, `'min'` и вычисленными значениями.

**Для функции unpack_and_call(num_list, num_dict):**
1.  Вызвать функцию `collect_stats`, передав ей:
    *   Элементы списка `num_list` в качестве позиционных аргументов (используя распаковку `*`).
    *   Пары ключ-значение из словаря `num_dict` в качестве именованных аргументов (используя распаковку `**`).
2.  Вернуть результат, полученный от `collect_stats`.

### Рекомендации
-   Вынести в отдельную функцию: Основную логику по расчету статистики необходимо вынести в функцию `collect_stats`. Функция `unpack_and_call` является демонстрационной оберткой для показа распаковки и должна быть предельно простой.
-   PEP8:
    -   Используй осмысленные имена переменных. Например, `all_numbers` для объединенного списка чисел, `stats` для итогового словаря.
    -   Если строка кода становится слишком длинной (особенно вызов функции с распаковкой), можно перенести аргументы на новую строку с отступом.
