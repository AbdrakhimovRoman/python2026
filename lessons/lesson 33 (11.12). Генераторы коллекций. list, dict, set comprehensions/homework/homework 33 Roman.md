# Домашнее задание. Урок 33. Генераторы коллекций. list, dict, set comprehensions

## Краткая сводка
- Основные ошибки на занятии: отсутствие dict comprehension, игнорирование фильтров по количеству шагов и длительности пайплайнов, неправильное форматирование строк.
- Цели ДЗ: закрепить list comprehension с форматом строки, научиться аккуратно суммировать значения и применять фильтры внутри dict/set comprehension, подготовиться к темам с обработкой логов.

## Задача 1. Повторение — «короткие имена»
- **Описание:** дан список `names = ["Анна", "Светлана", "Юра", "Даниил"]`. Сформируй список строк `"Имя — N букв/буквы"` только для имён длиной < 6. Для N=2–4 используй «буквы», иначе «букв». Формат разделителя — длинное тире с пробелами.
- **Критерии:** один list comprehension, корректное склонение и формат строки, исходный список не меняется.

## Задача 2. Практика — «компактные встречи»
- **Описание:** список `meetings = [("Демо", 2, 20), ("Планирование", 6, 45), ...]`, где кортеж — `(topic, members, minutes)`. Сформируй список строк `"<topic>: <members> чел, <minutes> мин"` только если `members < 5` и `minutes <= 30`.
- **Критерии:** один list comprehension, оба условия в конце, результат сохраняется в переменную `compact_meetings`, никаких `print` внутри comprehension.

## Задача 3. Интеграция — «статистика пайплайнов»
- **Описание:** список словарей `pipelines = [{"name": "ETL", "owner": "Антон", "steps": [...]}, ...]`, где каждый `steps` — список словарей с полями `duration` (мин) и `is_async`.
  - Построй словарь `pipeline_stats`, где ключ — название пайплайна, значение — словарь `{"owner": ..., "async_ratio": ...}`.
  - Включай только пайплайны с ≥3 шагами и суммарной длительностью ≤ 240.
  - `async_ratio` = доля асинхронных шагов, округлённая до двух знаков.
- **Критерии:** один dict comprehension, внутри используются генераторы для подсчётов, фильтры по количеству шагов и длительности обязательны.

## Задача 4. Подготовка — «лог задач»
- **Описание:** список строк `events = ["backend:deploy:45", "frontend:code-review:18", ...]`. Нужно:
  1. Построить множество `rapid_reviews` из департаментов, где действие `"code-review"` и длительность < 20.
  2. Построить словарь `team_load`, где ключ — департамент, значение — список всех длительностей его задач, но только если каждая запись имеет корректный формат (ровно две `:`).
- **Критерии:** используем set и dict comprehension, проверяем формат строки, `int(duration)` вызывается один раз на элемент, исходные данные не меняются.
