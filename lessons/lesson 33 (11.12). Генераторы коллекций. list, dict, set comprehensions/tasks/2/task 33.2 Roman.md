# Задание 33.2. Генераторы коллекций. list, dict, set comprehensions

## Блок урока
«List comprehensions» отрабатывает умение собирать список сразу из нескольких источников данных, сохраняя лаконичность формулы.

## Условие задачи
**Сложность**: <span style="color: #c62828">● сложный</span>. примерное время: 15–20 минут

Есть три списка одной длины:
- `tasks` — названия задач
- `owners` — имена ответственных
- `durations` — оценка времени в минутах

Нужно построить одним выражением list comprehension список строк `"<имя>: <задача> (X мин)"` только для тех позиций, где длительность больше медианы списка `durations`. При равенстве длительности медиане элемент тоже включается. Порядок должен соответствовать исходным спискам.

## Пример входных данных
```
tasks = ["Разбор логов", "Code review", "Деплой"]
owners = ["Анна", "Борис", "Вика"]
durations = [30, 55, 40]
```

## Пример выходных данных
```
['Борис: Code review (55 мин)', 'Вика: Деплой (40 мин)']
```

## Критерии успешного выполнения
- Использовано одно выражение list comprehension с одновременным обходом трёх списков.
- Медиана рассчитывается заранее и сравнивается с целыми числами.
- Формат строки строго `"Имя: Задача (N мин)"`.
- Порядок элементов сохраняет исходную последовательность.

## Подсказки
- Можно пройтись по zip трёх списков.
- Медиану для нечётной длины взять серединным значением отсортированного списка.

## Частые ошибки и ограничения
- Не добавляй вложенные циклы, достаточно одного прохода по zip.
- Следи, чтобы медиана считалась до comprehension, иначе код станет нечитаемым.
- Не модифицируй исходные списки во время решения.
