# Вопрос 1. Тема: Сложные запросы
  
**1. Какой оператор используется для объединения двух таблиц в SQL?**
  
Варианты ответов:
1) UNION
2) COMBINE
3) ✅ JOIN
4) MERGE
  
### ИИ анализ
Оператор `JOIN` является основным для объединения строк из двух или более таблиц на основе связанного столбца между ними.  \
`UNION` объединяет наборы результатов, а `COMBINE` и `MERGE` не являются стандартными SQL-операторами для этой цели.  \
Ответ верный и демонстрирует понимание ключевых операторов SQL для работы с таблицами.  
  
# Вопрос 2. Тема: Сложные запросы
  
**2. Как объединить таблицы Customers и Orders по полю CustomerID и выбрать все строки из таблицы Customers, даже если нет соответствующих записей в таблице Orders?**
  
Варианты ответов:
1) SELECT * FROM Customers FULL JOIN Orders ON Customers.CustomerID = Orders.CustomerID
2) SELECT * FROM Customers INNER JOIN Orders ON Customers.CustomerID = Orders.CustomerID
3) ✅ SELECT * FROM Customers LEFT JOIN Orders ON Customers.CustomerID = Orders.CustomerID
4) SELECT * FROM Customers RIGHT JOIN Orders ON Customers.CustomerID = Orders.CustomerID
  
### ИИ анализ
`LEFT JOIN` возвращает все записи из левой таблицы (Customers) и соответствующие записи из правой (Orders).  \
Если соответствия нет, результат для правой таблицы будет `NULL`. `INNER JOIN` вернет только совпадающие записи, а `RIGHT JOIN` — все записи из правой таблицы.  \
Выбран правильный вариант, который точно соответствует условию задачи.  
  
# Вопрос 3. Тема: Сложные запросы
  
**3. Что вернёт запрос `SELECT Customers.Name, Orders.OrderID FROM Customers LEFT JOIN Orders ON Customers.CustomerID = Orders.CustomerID;`, если в таблице Customers есть клиенты без заказов?**
  
Варианты ответов:
1) Запрос вернёт только клиентов с заказами
2) Запрос не вернёт результатов
3) Запрос вернёт только заказы, без клиентов
4) ✅ Запрос вернёт всех клиентов (включая тех, у кого нет заказов) с пустыми значениями для OrderID
  
### ИИ анализ
Поскольку используется `LEFT JOIN`, в результирующем наборе будут все строки из таблицы `Customers`.  \
Для клиентов без заказов не найдется соответствующей строки в `Orders`, поэтому в колонке `OrderID` будет отображено `NULL` (пустое значение).  \
Ответ верный и показывает понимание логики работы левого соединения.  
  
# Вопрос 4. Тема: Сложные запросы
  
**4. Какой результат вернёт следующий SQL-запрос?**

```
SELECT Name FROM Customers
UNION
SELECT Name FROM Suppliers;
```
  
Варианты ответов:
1) Вернёт только совпадающие значения из обеих таблиц
2) Вернёт только уникальные имена из таблиц Customers и Suppliers
3) Объединит результаты, но добавит NULL вместо отсутствующих значений
4) ❌ Объединит результаты обеих таблиц, включая дубликаты
  
### ИИ анализ
Оператор `UNION` по умолчанию объединяет результаты двух `SELECT`-запросов, автоматически удаляя дублирующиеся строки.  \
Правильным описанием результата является вариант 2: "Вернёт только уникальные имена из таблиц Customers и Suppliers".  \
Выбранный вариант неверен, так как он противоречит основному свойству `UNION` — исключению дубликатов.  
  
# Вопрос 5. Тема: Сложные запросы
  
**5. Какой результат вернёт следующий SQL-запрос?**

```
SELECT ProductName FROM Products
EXCEPT
SELECT ProductName FROM DiscontinuedProducts;
```
  
Варианты ответов:
1) Вернёт только устаревшие товары
2) ✅ Вернёт товары, которые есть в Products, но отсутствуют в DiscontinuedProducts
3) Вернёт только товары, которые присутствуют в обеих таблицах
4) Вернёт список всех товаров, включая устаревшие
  
### ИИ анализ
Оператор `EXCEPT` (или `MINUS` в некоторых СУБД) возвращает строки из первого набора данных, которые отсутствуют во втором.  \
Таким образом, запрос вернет имена продуктов, которые есть в таблице `Products`, но не числятся в таблице `DiscontinuedProducts`.  \
Ответ верный и демонстрирует понимание работы оператора для вычитания наборов данных.
