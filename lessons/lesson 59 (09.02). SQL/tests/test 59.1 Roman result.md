# Вопрос 1. Тема: Основы SQL
  
**1. Какая команда используется для извлечения всех записей из таблицы в SQL?**
  
Варианты ответов:
1) SELECT all FROM table_name;
2) SELECT FROM table_name;
3) SELECT 1 FROM table_name;
4) ✅ SELECT * FROM table_name;
  
### ИИ анализ
Правильный ответ выбран верно. Команда `SELECT * FROM table_name;` использует звездочку (`*`) как специальный символ, означающий "все столбцы". Это стандартный и наиболее распространенный способ извлечения всех данных из таблицы. Другие варианты синтаксически неверны или имеют иное назначение.

# Вопрос 2. Тема: Основы SQL
  
**2. Как ограничить выборку по конкретному условию?**
  
Варианты ответов:
1) ✅ SELECT * FROM table_name WHERE condition;
2) SELECT WHERE condition FROM table_name;
3) SELECT * FROM table_name ORDER BY condition;
4) SELECT * WHERE condition FROM table_name;
  
### ИИ анализ
Ответ правильный. Ключевое слово `WHERE` в SQL используется для фильтрации записей на основе указанного условия. Оно должно располагаться после оператора `FROM`. Варианты 2 и 4 нарушают правильный порядок предложений SQL, а вариант 3 использует `ORDER BY` для сортировки, а не для фильтрации.

# Вопрос 3. Тема: Основы SQL
  
**3. Как отсортировать результаты по убыванию значения в столбце age?**
  
Варианты ответов:
1) ✅ SELECT * FROM table_name ORDER BY age DESC;
2) SELECT * FROM table_name ORDER BY DESC age;
3) SELECT * FROM table_name WHERE age ORDER BY DESC;
4) SELECT * ORDER BY age FROM table_name;
  
### ИИ анализ
Вы ответили правильно. Для сортировки по убыванию используется ключевое слово `DESC` (descending), которое ставится после имени столбца в предложении `ORDER BY`. В других вариантах нарушен синтаксис: `DESC` стоит не на своем месте, отсутствует столбец для сортировки или нарушен порядок предложений SQL.

# Вопрос 4. Тема: Основы SQL
  
**4. Как выбрать записи, где age меньше 30 или salary больше 50000, и отсортировать их по name?**
  
Варианты ответов:
1) SELECT * FROM table_name ORDER BY age < 30 OR salary > 50000, name;
2) SELECT * FROM table_name WHERE age < 30 AND salary > 50000 ORDER BY name;
3) ✅ SELECT * FROM table_name WHERE age < 30 OR salary > 50000 ORDER BY name;
4) SELECT * FROM table_name WHERE age < 30 OR salary > 50000 name;
  
### ИИ анализ
Правильный ответ выбран верно. Для реализации условия "ИЛИ" используется логический оператор `OR`. Сортировка выполняется с помощью отдельного предложения `ORDER BY`. Вариант 1 пытается задать условие в `ORDER BY`, что неверно. Вариант 2 использует `AND`, что ужесточает условие. Вариант 4 пропускает ключевые слова `ORDER BY`.

# Вопрос 5. Тема: Основы SQL
  
**5. Как выбрать записи, где salary больше 50000 и age больше 25?**
  
Варианты ответов:
1) ❌ SELECT * FROM table_name WHERE salary > 30000, age > 25;
2) SELECT * FROM table_name WHERE salary > 30000 NOT age > 25;
3) SELECT * FROM table_name WHERE salary > 30000 OR age > 25;
4) SELECT * FROM table_name WHERE salary > 30000 AND age > 25;
  
### ИИ анализ
Выбранный ответ неверен. Для одновременного выполнения двух условий должен использоваться логический оператор `AND`. В выбранном варианте (1) используется запятая, что не является правильным синтаксисом SQL для объединения условий. Кроме того, в условии указано `salary > 30000`, что не соответствует условию задачи (`salary > 50000`). Правильным был бы вариант 4 с исправленным значением зарплаты.
