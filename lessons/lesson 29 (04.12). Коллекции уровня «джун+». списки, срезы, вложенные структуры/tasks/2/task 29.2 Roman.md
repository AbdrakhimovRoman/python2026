# Задание 29.2. Коллекции уровня «джун+». списки, срезы, вложенные структуры

## Блок урока
Индексация и срезы. Научимся извлекать конкретные диапазоны и копии списков, используя границы и шаг.

## Условие задачи
Ты ведешь список версий проекта, которые заливал в репозиторий. Нужно:
1. Показать первые три версии для краткой презентации.
2. Получить последние четыре версии, чтобы проверить актуальные изменения.
3. Собрать версии с индекса 2 по 7, чтобы оценить прогресс середины спринта.
4. Сделать копию списка без первой и последней версии для экспериментов.

## Пример входных данных
```python
builds = ["v0.1", "v0.2", "v0.3", "v0.4", "v0.5", "v0.6", "v0.7", "v0.8", "v0.9"]
```

## Пример выходных данных
```
Презентация: ['v0.1', 'v0.2', 'v0.3']
Свежие сборки: ['v0.6', 'v0.7', 'v0.8', 'v0.9']
Середина спринта: ['v0.3', 'v0.4', 'v0.5', 'v0.6', 'v0.7', 'v0.8']
Без краёв: ['v0.2', 'v0.3', 'v0.4', 'v0.5', 'v0.6', 'v0.7', 'v0.8']
```

## Критерии успешного выполнения
- Все выборки реализованы через срезы
- Присутствуют примеры с положительными и отрицательными границами
- Копия списка сделана через `[:]`, а не через методы
- Каждый результат снабжен пояснением в выводе

## Подсказки
- Чтобы получить хвост, используй отрицательные индексы `[-4:]`
- Диапазон `[2:8]` вернёт элементы с индекса 2 по 7
- `builds[1:-1]` удаляет крайние элементы
- Пустая граница слева или справа облегчает запись начала или конца

## Частые ошибки и ограничения
- Не меняй исходный список
- Не указывай шаг 0
- Следи, чтобы границы не выходили за длину списка
- Не забывай, что `stop` не включается, если нужно включить элемент — увеличивай его на 1
