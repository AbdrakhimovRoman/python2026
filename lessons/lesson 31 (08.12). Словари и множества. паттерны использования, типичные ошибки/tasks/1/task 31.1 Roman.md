# Задание 31.1. Словари и множества. паттерны использования, типичные ошибки
## Блок урока
Словарь как структура быстрого доступа — Блок показывает, как превращать повторяющиеся данные в словарь и мгновенно находить значения по ключу.

## Условие задачи
Добровольческий кол-центр хранит, какой оператор последний перезвонил клиенту. Координатору нужно быстро понять, кто отвечает за сложный запрос. Создай словарь last_operator, где ключ — номер обращения, а значение — имя оператора. Затем выведи информацию для переменных current_ticket и stale_ticket.

## Пример входных данных
```python
callbacks = [
    ('ticket-101', 'Степан'),
    ('ticket-204', 'Мария'),
    ('ticket-350', 'Юля'),
]
current_ticket = 'ticket-204'
stale_ticket = 'ticket-999'
```

## Пример выходных данных
```text
ticket-204 -> Мария
ticket-999 -> ожидает назначения
```

## Критерии успешного выполнения
- Список пар преобразован в словарь last_operator с ключом 'номер обращения'.
- Для обращения к словарю используется метод get с значением по умолчанию.
- Выведены ответы для current_ticket и stale_ticket в заданном формате.
- Код не падает при запросе отсутствующего ключа.

## Подсказки
- Пройдись по списку callbacks в цикле и наполняй словарь last_operator.
- Используй множественное присваивание: key, value = запись из callbacks.
- Для отсутствующих данных возвращай строку 'ожидает назначения'.

## Частые ошибки и ограничения
- Не обращайся к словарю через квадратные скобки, если нет проверки на существование ключа.
- Не оставляй ключи кортежами целиком — нужен именно идентификатор.
- Не дублируй вывод: достаточно одной строки на каждый запрос.
